◆ cy_afe_feed()

cy_rslt_t cy_afe_feed ( cy_afe_t  handle,
CY_AFE_DATA_T *  input_buffer,
CY_AFE_DATA_T *  aec_ref 
)

Feed the audio data to audio front end middleware (10ms worth of data)

Memory for the input buffer & aec reference data (if AEC is enabled) need to be allocated by application. Application shall not free/reuse input buffer & AEC reference buffer till AFE middleware provides cy_afe_output_callback_t callback. Application may have choice to handle (free/reuse) the input & aec reference buffers on cy_afe_output_callback_t callback.

If Application needs to feed stereo data, then the data format must be non-interleaved format. example: For stereo data of 10ms worth, first 320bytes must be of channel-1 and then next 320bytes must be channel-2. application must pass enough number of bytes, if it doesn't pass then AFE middleware can go into unexpected state.

AEC reference data shall be mono only.

Parameters
[in]handleHandle to audio front end instance created by the cy_afe_create API
[in]input_bufferPointer to input buffer to pass the audio data
[in]aec_refAEC reference mono audio data
Returns
CY_RSLT_SUCCESS on success; an error code on failure.
